# Feature Build

Use this skill when adding new features to the app.

## Rules
- First understand the current structure
- Make minimal changes only
- Reuse existing components and hooks
- Do NOT rewrite unrelated code
- Keep UX fast and simple

## Product Thinking
- Every feature must improve speed, clarity, or usability
- Avoid unnecessary complexity
- Prefer simple solutions over perfect architecture

## Implementation Rules
- Keep logic inside hooks (e.g. useWorkout)
- Keep UI components clean and focused
- Avoid putting too much logic into page.tsx

## Workflow
1. Analyze current implementation
2. Identify where the feature fits
3. Explain minimal plan
4. Implement feature
5. Validate behavior
